Module: BasicDisplay

Renders a basic digital display. This module provide APIs for changing the look and feel of the rendered text, including: cursors, background color, font, size, alignment.

Example

Example use of the widget.

 // Example pvsio-web demo that uses BasicDisplay
 // The following configuration assumes the pvsio-web demo is stored in a folder within pvsio-web/examples/demo/
 require.config({
     baseUrl: "../../client/app",
     paths: {
         d3: "../lib/d3",
         lib: "../lib"
     }
 });
 require(["widgets/BasicDisplay"], function (BasicDisplay) {
      "use strict";
      var device = {};
      device.disp = new BasicDisplay("disp", {
        top: 100, left: 120, height: 24, width: 120
      }, {
        displayKey: "disp"
      });
      device.disp.render( { disp: 10.5 }); // the display renders 10.5
 });

Methods

BasicDisplay(id, coords, opt)

Constructor.
Parameters:
Name Type Description
id String The ID of the display.
coords Object The four coordinates (top, left, width, height) of the display, specifying the left, top corner, and the width and height of the (rectangular) display. Default is { top: 0, left: 0, width: 200, height: 80 }.
opt Object Options:
  • displayKey (string): the name of the state attribute defining the display content. This information will be used by the render method. Default is the ID of the display.
  • fontfamily (String): font type (default is "sans-serif")
  • fontsize (Number): font size (default is 0.8 * height)
  • fontfamily (String): font family, must be a valid HTML5 font name (default is "sans-serif")
  • fontColor (String): font color, must be a valid HTML5 color (default is "white", i.e., "#fff")
  • backgroundColor (String): background display color (default is black, "#000")
  • align (String): text alignment (available options are "left", "right", anc "center". Default is "center")
  • letterSpacing (Number): spacing between characters, in pixels (default is 0)
  • inverted (Bool): if true, the text has inverted colors, i.e., fontColor becomes backgroundColor, and backgroundColor becomes fontColor (default is false)
  • borderWidth (Number): border width (default is 0, i.e., no border)
  • borderStyle (String): border style, must be a valid HTML5 border style, e.g., "solid" (default is "none")
  • borderColor (String): border color, must be a valid HTML5 color (default is "black")
  • cursor (String): cursor style, must be a valid HTML5 cursor style, e.g., "pointer", "crosshair", etc. (default is "default")
  • blinking (Bool): true means the text is blinking (default is false, i.e., not blinking)
  • format (String): sets the display format. When this option is set to "mm:ss", the display value represents seconds, and format to be displayed is mm:ss
  • parent (String): the HTML element where the display will be appended (default is "body")

    toJSON()

    Returns a serialised version of the widget in JSON format. This is useful for saving/loading a specific instance of the widget. In the current implementation, the following attributes are included in the JSON object:
  • type (string): widget type, i.e., "display" in this case
  • id (string): the unique identifier of the widget instance
  • fontsize (string): the font size of the display
  • fontColor (string): the font color of the display
  • backgroundColor (string): the background color of the display
  • displayKey (string): the name of the state attribute defining the display content.
  • visibleWhen (string): a booloan expression defining when the condition under which the widget is visible
  • auditoryFeedback (string): whether display readback is enabled
